# TODOリスト
## 主な機能
### ユーザー登録・認証機能
Firebase Authenticationを使用してユーザー認証を行う
- ユーザー登録・ログイン・ログアウトはクライアント側でFirebase SDKを使用
- バックエンドはFirebase ID Token（JWT）を検証し、ユーザーを識別する
- 初回アクセス時にFirebase UIDをもとにuserテーブルにレコードを自動作成する
### TODO登録機能
- タイトル
- 内容
- 期日
- 完了したかどうか
### TODO削除機能
### TODO編集機能
### TODO閲覧機能
### メール通知機能
- 期日を超えた、または３日以内になった場合にメールに通知する機能
- 通知種別
  - approaching: 期日3日以内（0日含む）
  - overdue: 期日超過（overdueの場合はapproachingは送らない）
- 1日1回送信（同日に同一TODOへの重複送信は防止）
- バッチ処理で定期的にチェック（1日1回想定）

## DB設計

### userテーブル
| カラム名 | 型 | NULL | キー | デフォルト | 説明 |
|----------|-----|------|------|------------|------|
| id | INT | NO | PK | AUTO_INCREMENT | ユーザーID |
| firebase_uid | VARCHAR(128) | NO | UNIQUE | - | Firebase UID |
| email | VARCHAR(255) | NO | UNIQUE | - | メールアドレス |
| name | VARCHAR(100) | NO | - | - | ユーザー名 |
| created_at | DATETIME | NO | - | CURRENT_TIMESTAMP | 作成日時 |
| updated_at | DATETIME | NO | - | CURRENT_TIMESTAMP ON UPDATE | 更新日時 |
| deleted_at | DATETIME | YES | - | NULL | 削除日時(論理削除) |

### todoテーブル
| カラム名 | 型 | NULL | キー | デフォルト | 説明 |
|----------|-----|------|------|------------|------|
| id | INT | NO | PK | AUTO_INCREMENT | TODO ID |
| user_id | INT | NO | FK(user.id) | - | ユーザーID |
| title | VARCHAR(255) | NO | - | - | タイトル |
| content | TEXT | YES | - | NULL | 内容 |
| due_date | DATE | YES | - | NULL | 期日 |
| is_completed | BOOLEAN | NO | - | FALSE | 完了フラグ |
| created_at | DATETIME | NO | - | CURRENT_TIMESTAMP | 作成日時 |
| updated_at | DATETIME | NO | - | CURRENT_TIMESTAMP ON UPDATE | 更新日時 |
| deleted_at | DATETIME | YES | - | NULL | 削除日時(論理削除) |

### notificationテーブル
| カラム名 | 型 | NULL | キー | デフォルト | 説明 |
|----------|-----|------|------|------------|------|
| id | INT | NO | PK | AUTO_INCREMENT | 通知ID |
| todo_id | INT | NO | FK(todo.id) | - | TODO ID |
| user_id | INT | NO | FK(user.id) | - | ユーザーID |
| type | VARCHAR(32) | NO | - | - | 通知種別('approaching', 'overdue') |
| sent_at | DATETIME | NO | - | CURRENT_TIMESTAMP | 送信日時 |

### ER図
```
user 1 ---- * todo
user 1 ---- * notification
todo 1 ---- * notification
```

### インデックス
- user: firebase_uid (UNIQUE), email (UNIQUE)
- todo: user_id
- notification: todo_id, user_id

※ 性能問題が発生した場合に複合インデックス追加を検討

### 外部キー制約
- todo.user_id -> user.id (ON DELETE RESTRICT)
- notification.todo_id -> todo.id (ON DELETE CASCADE)
- notification.user_id -> user.id (ON DELETE RESTRICT)

## API設計

### 認証
全てのAPIはFirebase認証が必要（AuthorizationヘッダーにFirebase ID Token を Bearer トークンとして設定）
- バックエンドはFirebase Admin SDKを使用してID Tokenを検証する
- トークンからFirebase UIDを取得し、userテーブルと紐付けてユーザーを識別する

### TODO API

#### TODO一覧取得
- **エンドポイント**: GET /todos
- **認証**: 必須
- **レスポンス**:
```json
{
  "todos": [
    {
      "id": 1,
      "user_id": 1,
      "title": "タイトル",
      "content": "内容",
      "due_date": "2025-01-31",
      "is_completed": false,
      "created_at": "2025-01-01T00:00:00Z",
      "updated_at": "2025-01-01T00:00:00Z"
    }
  ]
}
```

#### TODO登録
- **エンドポイント**: POST /todos
- **認証**: 必須
- **リクエストパラメータ**:

| パラメータ | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| title | string | YES | タイトル（最大255文字） |
| content | string | NO | 内容 |
| due_date | datetime | NO | 期日（ISO 8601形式） |

- **リクエスト例**:
```json
{
  "title": "買い物に行く",
  "content": "牛乳と卵を買う",
  "due_date": "2025-01-31T00:00:00Z"
}
```
- **レスポンス**: 201 Created
```json
{
  "id": 1,
  "user_id": 1,
  "title": "買い物に行く",
  "content": "牛乳と卵を買う",
  "due_date": "2025-01-31",
  "is_completed": false,
  "created_at": "2025-01-01T00:00:00Z",
  "updated_at": "2025-01-01T00:00:00Z"
}
```
